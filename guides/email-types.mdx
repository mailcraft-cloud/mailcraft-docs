# Email Types

The `type` field tells AI what kind of email to write. It's a free-form string ‚Äî you don't need to register or configure types.

## Common types

| Type | Use case |
|------|----------|
| `welcome` | New user onboarding |
| `password-reset` | Password reset link |
| `otp-code` | Verification codes |
| `invoice` | Billing notifications |
| `trial-ending` | Trial expiration reminders |
| `order-shipped` | Shipping confirmations |
| `account-deleted` | Account deletion confirmations |
| `team-invite` | Team invitation emails |

## Tips

- **Use descriptive names** ‚Äî `order-shipped` is better than `notification`
- **Use hyphens** ‚Äî `trial-ending` not `trialEnding`
- **Be specific** ‚Äî `payment-failed` is better than `billing`
- **Pair with data** ‚Äî the type tells AI the intent, data provides the specifics
